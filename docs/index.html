PM2, but for Go services — with real zero-downtime

Zero-Downtime
Process Manager for Go

gpm ships new versions of your Go services with zero dropped connections — rolling reloads, cluster mode, and auto-reload-on-rebuild, in a single static binary. No Node.js, no config files.

Get started → View on GitHub
Install (macOS / Linux)
$ curl -fsSL https://raw.githubusercontent.com/parichit13/gpm/main/install.sh | bash

Downloads the right binary, verifies its checksum, installs to ~/.gpm/bin, and starts the daemon. Update anytime with gpm update.

// what you get

Built for production, not just &

Everything you'd reach for to keep a service available through deploys, crashes, and reboots.

♻️

Zero-downtime reload

Rolling, health-gated reloads. A new instance must pass its health check before the old one drains its in-flight requests — the port is never down.

⛓️

Cluster mode

Run N instances behind one port with -i N, and scale up or down live without dropping traffic.

👁️

Watch mode

Pass --watch and gpm auto-runs a zero-downtime reload whenever the binary changes. Just rebuild — traffic rolls over on its own.

🔌

Any binary

Use the tiny Go SDK (SO_REUSEPORT) for services you control, or the built-in TCP proxy for opaque binaries you can't change.

🛟

Crash-loop safe

Auto-restart with exponential backoff, an honest errored state, and exit-reason logging — so a flapping binary never hides as "stopped".

💾

Save & resurrect

gpm save then gpm resurrect brings your whole process list back — instances, ports, modes, health and all — after a reboot.

⬆️

Self-updating

gpm update fetches the latest release, verifies the checksum, swaps the binary in place, and restarts the daemon.

⚡

Zero-config daemon

The background daemon starts automatically on first use and after reboot. Services get stable integer ids you can use in any command.

🪶

One static binary

No Node.js, no runtime, no config files. A single Go binary and a Unix socket. Works on macOS and Linux, amd64 and arm64.

// how it works

    
Two ways to share the port

    
Zero-downtime means old and new instances both serve during the swap. gpm does that two ways, depending on whether you can modify the service.

    

      

        
reuseport default · SDK

        
For Go services you control. Import the ~3-line SDK; instances bind the same port with SO_REUSEPORT and the kernel load-balances. gpm probes a per-instance health endpoint to confirm the new one before draining the old.

        
    
          
  • ln, _ := gpm.Listen(":8080")
    • 
          
  • gpm.Serve(srv, ln) — graceful SIGTERM drain
    • 
          
  • No extra hop — traffic goes straight to your service
    • 
        

      

      

        
proxy opaque binaries

        
For binaries you can't change. gpm owns the public port and runs a built-in TCP load balancer, starting each instance on a private port it reads from $PORT. On reload it swaps upstreams atomically — retrying past a draining backend.

        
    
          
  • No code changes — just reads $PORT
    • 
          
  • Even round-robin across instances
    • 
          
  • Readiness gates traffic to a new instance
    • 
        

      

    

  







  

    
// quick start

    
From install to zero-downtime in four steps

    
The daemon auto-starts, so there's nothing to configure. Every command takes an id or a name.

    

      
01
gpm start ./api api \
  --port 8080 -i 3 --watch
Run 3 instances behind port 8080, watching the binary.

      
02
gpm list
See ids, status, instance counts, mode, port and uptime.

      
03
gpm reload api
# or just rebuild
Zero-downtime rollover. With --watch, a rebuild triggers it automatically.

      
04
gpm save
Persist the list so gpm resurrect restores it after a reboot.
